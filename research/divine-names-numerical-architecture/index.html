Research question

What happens when the source forms are treated as exact textual objects first?

The project asks what structure appears when divine names, titles, formulas, canonical totals, and Christian-comparative witnesses are checked under explicit source rules before any interpretation is attempted. The calculation pipeline is deliberately conservative: establish the object, compute the value, classify the pattern, then decide what the pattern can and cannot say.

Governing rule

Be exact about what happened, candid about why it is interesting, and open about what it means.

Method architecture

Source object first, arithmetic second, interpretation third.

The working paper separates textual status from numerical behavior so that biblical names, later traditional titles, Christian-comparative forms, New Testament references, formula labels, and rejected inherited rows are not silently mixed together.

01

Lock the source form

Identify the consonantal form, textual context, and corpus class before computing.

02

Compute declared values

Apply standard gematria, Mispar Gadol when relevant, and explicit base conversion.

03

Classify structure

Record digit sums, collapse paths, palindrome status, shared values, and canonical contacts.

04

Run controls

Use matched phrase controls, alternate-anchor audits, sensitivity checks, and finite enumerations.

Controls and limits

The project is intentionally not a one-line proof of meaning.

The matched-control study reports elevated primary base-12 structure, but not a statistically decisive result for the frozen primary biblical corpus. The Christian-inclusive layer is stronger and remains an exploratory result for future testing. The paper leaves theological significance open to the reader after the calculations and controls have been made inspectable.

What is separated

  • calculated values
  • textual source class
  • statistical control behavior
  • interpretive or theological meaning
  • exploratory echoes and future tests
